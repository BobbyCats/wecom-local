# WeCom Local CLI v0.1.1

Patch release focused on safer Agent defaults and release hardening.

## Highlights

- Added `auth status` and `auth prepare` so Agents can check or warm macOS
  Runtime Authorization without storing passwords.
- Changed `members` to return basic member fields by default.
- Added explicit `members --full` opt-in for sensitive locally visible profile
  fields such as account names, email addresses, phone numbers, and external
  identifiers.
- Hardened Conversation Reference ambiguity errors so they report counts and
  recovery guidance without listing candidate conversation names or ids.
- Hardened Runtime Bridge cleanup so LLDB scripts and runtime JSON export files
  are removed after attach attempts.
- Extended `store-probe --json` proof output with redacted format, key, page
  validation, and privacy fields while keeping Local Store Reader disabled.

## Safety Boundary

- Reads only data already visible to the signed-in macOS WeCom Desktop account.
- Does not send messages or write back to WeCom Desktop.
- Does not store local macOS passwords.
- Public docs, tests, and examples use synthetic data only.
- Local Store Reader remains proof-only and is not enabled in this release.
