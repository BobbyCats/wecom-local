# WeCom Local CLI v0.1.0

First public release of WeCom Local CLI, a local read-only WeCom Desktop query
CLI for humans and AI agents.

## Highlights

- Stable JSON commands for Agent workflows:
  - `doctor`
  - `conversations`
  - `history`
  - `members`
  - `search`
  - `stats`
  - `export`
  - `store-probe`
- Conversation Reference support: pass either a conversation id or a unique
  display-name query.
- Fail-closed ambiguity handling when a display-name query matches multiple
  conversations.
- Group member listing through locally visible WeCom Desktop runtime data.
- Member Participation aggregates through `stats --include-members`.
- Redacted Local Store Probe for file-format evidence before any future Local
  Store Reader work.
- macOS Runtime Authorization documentation for `sudo` and optional Touch ID.

## Runtime Attach Improvement

`stats --include-members` now reads history and members in one Runtime Bridge
attach instead of two. This reduces repeated attach churn for the common
analysis path while keeping the Runtime Bridge read-only.

## Safety Boundary

- Reads only data already visible to the signed-in macOS WeCom Desktop account.
- Does not send messages or write back to WeCom Desktop.
- Does not store local macOS passwords.
- Public docs, tests, and examples use synthetic data only.
- Local Store Reader is not implemented in this release.

## Tested Environment
